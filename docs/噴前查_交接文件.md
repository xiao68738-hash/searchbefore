更新日期：2026-07-15

## 專案概況

噴前查是給台灣農友使用的 PWA，功能包括農藥登記查詢、稀釋計算、施藥紀錄、安全採收期倒數，以及栽培、施肥、採收、採後處理、資材購入等田間作業紀錄。

- GitHub：https://github.com/xiao68738-hash/searchbefore
- 正式網站：https://xiao68738-hash.github.io/searchbefore/
- 農藥資料來源：農業部農藥開放資料

## 不可破壞的安全規則

1. 粉劑以公克 `g` 顯示，絕不可換算成瓶蓋、茶匙或湯匙。
2. 只有液劑 `ml` 可以顯示容量估算，實際配製仍以量具與標示為準。
3. 採收備註若列出露天、設施或不同用途天數，在沒有栽培方式選項前採最長天數。
4. 同一田區或種植批次的每一筆施藥都必須納入計算，取最晚可採日期。
5. 任一筆施藥紀錄缺少採收期時，該批次不得顯示「可採收」。
6. `track`、`notify` 或畫面篩選只能控制提醒與顯示，不能把實際施藥排除於安全計算。
7. 作物群組 `PARENT` 尚未逐項完成官方校驗，目前不可自動合併上層群組登記。
8. 採收事件即使觸發安全警告仍可留存，但不得省略警告與判定快照。

## v1.3.0 安全基礎

- 加入首頁醒目安全提醒。
- 粉劑移除瓶蓋與湯匙換算。
- 新增備註採收期解析，設施栽培採較長天數。
- 原始 `phi` 缺值但備註有採收天數時，可用保守天數建立紀錄。
- 採收期完全缺值時仍可保存施藥紀錄，但不顯示可採日，也不建立通知。
- 倒數改為依田區／種植批次彙總所有施藥紀錄。
- 舊手機通知按單筆施藥排程，可能早於同田區後續施藥的最晚日期，因此暫停自動通知；使用者可從安全彙總後的倒數卡加入行事曆。
- 加入紀錄 ID、`SCHEMA_VERSION`、舊資料備份及遷移。
- 暫停未完成校驗的作物群組聯集。

## 本次 v1.5.0 現有功能串接

- 田區詳情改為「完整作業時間軸」，用藥與五類農務依日期合併顯示。
- 新增「作業鏈概況」，顯示用藥、栽培、施肥、採收、採後與資材購入的已留存筆數。
- 概況僅表示有無紀錄，不計算合規分數，也不宣稱通過產銷履歷或有機驗證。
- 採收表單即時連動同田區所有用藥紀錄，顯示可採、等待或資料不足狀態。
- 若採收日早於最晚安全採收日，或有用藥缺少採收期，儲存前會要求確認；為保留事實紀錄，確認後仍可保存。
- 採收紀錄保存當時的安全判定快照，後續可追溯當時是安全、等待、未知或沒有連動資料。
- 新增完整作業 CSV，把用藥及所有農務依時間順序匯出；原本的農務專用 CSV 仍保留。
- 擴充測試 harness，涵蓋採收安全連動、整合時間軸、作業鏈概況與完整 CSV。

## 執行測試

```text
node tests/run-all.js
```

所有測試通過後，才可以合併到 `main`。

## GitHub 保存與發布流程

1. `main` 永遠保留可公開使用的穩定版。
2. 從 `main` 建立一個工作分支，例如 `feature/connected-records`。
3. 在工作分支完成修改與測試。
4. 保存成有意義的 Commit。
5. 建立草稿 Pull Request，檢查差異及測試結果。
6. 確認後合併到 `main`，GitHub Pages 才會更新正式網站。
7. 網站確認正常後，刪除已完成的工作分支並建立版本標籤。

## 授權私鑰

- `private_key.pem` 只能保存在開發者控制的加密備份中，絕不可上傳 GitHub、雲端公開連結或通訊軟體。
- `.gitignore` 已排除 `*.pem`，但發布前仍須人工檢查。
- 建議記錄公鑰指紋、建立日期與輪替方式；私鑰疑似外流時必須更換金鑰對及 App 內公鑰。
- 授權碼產生器若不含私鑰或其他祕密，可以版本化；加入 GitHub 前先人工檢查內容。

## 現行資料模型與後續方向

- `fieldPlots`：田區、作物、種植日期與批次。
- `records`：每次實際施藥；同批次全部紀錄共同決定最晚可採日。
- `farmRecords`：栽培、施肥、採收、採後、資材購入；採收可帶 `safetyCheck` 快照。

下一階段先透過農友試用確認實際欄位與操作順序，再考慮投入批次追溯關係、附件／憑證、庫存扣抵、團隊權限，以及農糧署或驗證機構正式格式轉換。官方匯入規格未確認前，不應宣稱能直接申報或送驗。
