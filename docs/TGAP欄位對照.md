# TGAP 生產紀錄欄位對照表

更新日期：2026-07-17

## 目的
盤點噴前查 App 現有紀錄欄位，對照產銷履歷 TGAP「生產紀錄」應登錄之欄位，找出未來若要透過農糧署 API（L4→L3）上傳時的缺口。此文件僅供內部規劃，**不代表 App 已可申報或上傳**。

## 依據與範圍
- 依據：《產銷履歷農產品生產過程 臺灣良好農業規範(TGAP)-雜糧類》紀錄簿通用格式（作為代表；**各作物 TGAP 略有差異，實際以主打作物之最新版為準**）。
- 資料來源：[TGAP 下載 - 產銷履歷農產品資訊網](https://taft.moa.gov.tw/lp-1047-1.html)、[農糧署 TGAP 管理規範](https://www.afa.gov.tw/cht/index.php?code=list&ids=3418)。
- API 傳輸方式（凌網 2026-07-17 回覆）：Web Service（SOAP）+ XML，欄位以**官方代碼**為主，代碼表與介接規格須待農糧署審核通過後提供。

## 缺口與優先順序

### A. 現在就能補（簡單、低風險，不需等 API 規格）
1. **用藥紀錄補「操作人員」欄位** — TGAP 表7 明列必填，目前 records 沒有。
2. **施肥「施肥別（基肥/追肥）」改為選項** — 目前併在自由文字 method。
3. **田區補「品種」欄位** — 目前只有作物。
4. （選配）用藥補「使用量/施藥面積」、施肥液肥補「稀釋倍數」。

### B. 真正的工作量（需等審核通過拿到規格才動工）
5. **官方代碼系統** — 作物、品種、作業內容、肥料/防治/其他資材、防治對象，全部要從**自由文字 → 官方代碼**。代碼表本身要透過 API（作物代碼下載、品種資料下載、資材下載…）取得，**審核通過才拿得到**。這是整個串接最大的一塊，現在不要憑猜測改資料庫。
6. **SOAP 後端代理** — 純前端打不了 L3 主機，需一個後端做認證與 XML 對應（同樣待審核）。

### C. 可暫不處理
7. 資材入出庫/庫存結存（表6/9/12）、地段地號/面積、種子來源、包裝出貨、設備保養等——依農友試用回饋再決定是否投入。

## 建議
- 先做 A 區三項（操作人員、施肥別、品種），讓資料「未來可對應」，同時不誤導使用者。
- B 區等 Hyweb 回覆（每位農友是否各自驗證、能否先取得沙箱/WSDL）與審核結果後再排。
- 對照範圍以**實際主打作物**的最新版 TGAP 再核對一次（本表以雜糧類通用格式為代表）。
