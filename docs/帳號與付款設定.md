# Google 登入與自願贊助設定

## 目前採用的原則

- Google 登入是選用功能，不登入仍可使用目前所有功能及建立本機紀錄。
- Google 登入目前只用於識別帳號。
- 田區、用藥、施肥、採收等農務紀錄不會因登入而自動上傳。
- 公開測試期間不顯示方案、購買、授權碼或試用申請入口。
- 咖啡贊助完全自願，是否贊助都不影響功能或服務。

## 啟用 Google 登入

1. 到 Firebase Console 建立專案。
2. 在專案中新增「Web 應用程式」。
3. 到 Authentication 啟用 Google 登入方式。
4. 在 Authentication 的 Authorized domains 加入 `xiao68738-hash.github.io`。
5. 將 Firebase 提供的 Web 設定物件填入根目錄 `service-config.js` 的 `firebase`。
6. 同步更新 `sw.js` 的 `CACHE_VERSION`，確保使用者取得新設定。
7. 發布後只在正式 HTTPS 網址測試；直接開啟本機 `file://` 預覽時不執行登入。

Firebase 的 Web 設定物件是公開識別設定，不是管理員私鑰；任何私鑰或服務帳戶 JSON 都不得放進本專案或 GitHub。

## 啟用贊助連結

1. 在贊助平台建立自己的公開頁面。
2. 確認網址使用 `https://`。
3. 將完整網址填入 `service-config.js` 的 `supportUrl`。
4. 贊助頁應清楚說明贊助是自願支持，不影響任何功能或服務。
5. 未來封裝成 Google Play App 前，需再次核對當時的外部連結政策。

若未填入網址，個人頁只會顯示「贊助連結準備中」，不會導向任何外部頁面。

## 未來若加入 Google Drive 備份

- 優先採用使用者自己的 Google Drive，並只申請完成備份所需的最小權限。
- 啟用前必須更新隱私權政策，說明存取範圍、刪除方式與停止授權的方法。
- 不把田間資料另存到開發者控制的資料庫，除非使用者另外明確同意。
