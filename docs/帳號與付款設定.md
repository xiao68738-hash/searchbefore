# Google 登入與自願贊助設定

## 目前採用的原則

- Google 登入是選用功能，不登入仍可使用目前所有功能及建立本機紀錄。
- Google 登入目前只用於識別帳號。
- 田區、用藥、施肥、採收等農務紀錄不會因登入而自動上傳。
- 公開測試期間不顯示方案、購買、授權碼或試用申請入口。
- 咖啡贊助完全自願，是否贊助都不影響功能或服務。

## 公開測試流程

1. 先用網頁版招募少量真實農友，不必等 Google Play 正式上架。
2. 請測試者完成一項平常真的會做的工作：查詢、建立田區、留下紀錄。
3. 邀請測試者隔幾天回來查看採收倒數或田區時間軸，再提供回饋。
4. 觀察完成紀錄與再次回訪的人數，不以按讚、留言或單次開啟判斷需求。
5. 累積穩定測試者後，再邀請其中的 Android 使用者參加 Google Play 封閉測試。

公開曝光前，請在 `service-config.js` 填入 `feedbackEmail`。若尚未設定，回饋按鈕只會協助使用者複製內容，不會寄往範例地址。

## 啟用 Google 登入

1. 到 Firebase Console 建立專案。
2. 在專案中新增「Web 應用程式」。
3. 到 Authentication 啟用 Google 登入方式。
4. 在 Authentication 的 Authorized domains 加入 `searchbefore.tw` 與 `www.searchbefore.tw`；遷移期間保留 `xiao68738-hash.github.io`。
5. 將 Firebase 提供的 Web 設定物件填入根目錄 `service-config.js` 的 `firebase`。
6. 同步更新 `sw.js` 的 `CACHE_VERSION`，確保使用者取得新設定。
7. 發布後只在正式 HTTPS 網址測試；直接開啟本機 `file://` 預覽時不執行登入。

Firebase 的 Web 設定物件是公開識別設定，不是管理員私鑰；任何私鑰或服務帳戶 JSON 都不得放進本專案或 GitHub。

## 啟用贊助連結

1. 在贊助平台建立自己的公開頁面。
2. 確認網址使用 `https://`。
3. 將完整網址填入 `service-config.js` 的 `supportUrl`。
4. 贊助頁應清楚說明贊助是自願支持，不影響任何功能或服務。
5. 未來封裝成 Google Play App 前，需再次核對當時的外部連結政策。

若未填入網址，首頁與個人頁都不顯示贊助區塊；不會出現未完成的按鈕或導向任何外部頁面。

## 未來付費功能的界線

建議持續開放合法用藥查詢、安全採收資訊、基本換算、基本紀錄，以及使用者自己資料的查看、備份與匯出。這些功能與安全判斷及資料自主權直接相關，不適合作為付費門檻。

未來較適合收費的是額外節省時間或產生成本的服務，例如跨裝置雲端同步、自動提醒、語音轉紀錄、複雜文件整理、客製報表，以及多人或多農場協作。正式啟用任何付費功能前，再依網站與 Google Play 當時適用的付款規則分開處理。

## 未來若加入 Google Drive 備份

- 優先採用使用者自己的 Google Drive，並只申請完成備份所需的最小權限。
- 啟用前必須更新隱私權政策，說明存取範圍、刪除方式與停止授權的方法。
- 不把田間資料另存到開發者控制的資料庫，除非使用者另外明確同意。

## Firebase Hosting 正式發布

1. 執行 `npm install` 安裝發布工具。
2. 執行 `npm run release:check`，確認測試與 `dist/` 成品檢查全部通過。
3. 第一次使用先執行 `npm run firebase:login`，再執行 `npm run deploy:preview` 取得暫時預覽網址。
4. 預覽確認後執行 `npm run deploy:hosting`，發布到 Firebase 正式預設網址測試。
5. 確認查詢、離線、Google 登入、備份與回饋功能後，再於 Firebase Hosting 連接 `searchbefore.tw`。
6. 依 Firebase 畫面提供的值修改 GoDaddy DNS；不可同時保留指向 GitHub Pages 的 A/CNAME 紀錄。
7. 自訂網域與 HTTPS 正常後，才停用 GitHub Pages並將 GitHub 專案改為私人。

Firebase 只部署 `dist/`；`docs/`、`tests/`、README、建置腳本與 Git 歷史不會放進公開網站。
