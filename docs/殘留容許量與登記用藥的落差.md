# 殘留容許量(MRL)與登記用藥的落差

更新日期:2026-07-19
狀態:**設計文件,尚未實作**。目前 App 只呈現「登記」資料,不含任何 MRL 資訊。
起因:農友回饋實例——「有些藥劑農藥所標準能用,但衛福部不能用」,虎豆即為實例;於市場或北農驗到即直接銷毀。

## 一、問題本質:兩個機關、兩套標準

| | 主管機關 | 管的事 | 對農友的意義 |
|---|---|---|---|
| **農藥登記** | 農業部防檢署 | 這支藥**可不可以用**在這個作物 | 合法施用 |
| **殘留容許量(MRL)** | 衛福部食藥署 | 收穫物上**可以殘留多少** | 產品能不能通過檢驗 |

兩者**各自獨立**。因此存在一種農友最容易踩到的狀況:

> **合法登記可用,但殘留標準未訂 → 依法即「不得檢出」→ 市場抽驗一驗到就銷毀。**

法源:《農藥殘留容許量標準》明定「**表中未列者均不得檢出**」。也就是說,MRL 表沒有列到的「藥劑×農產品」組合,標準是零,不是「沒規定所以沒事」。

農友的直覺通常是「登記可以用 = 用了沒問題」,這正是落差所在;而現行 App **只顯示了登記這一半**,等於在無意間強化了這個誤解。

### 為什麼會發生
- 登記時的作物分類與 MRL 的**農產品分類不一定對齊**(例:青蒜屬小葉菜類、蒜頭屬根莖菜類——見 `docs/作物收穫型態消歧.md`)。
- 新登記或擴大使用範圍後,MRL 訂定有時間差。
- 小宗作物(如虎豆)常無對應 MRL。

**前置條件**:MRL 是按農產品分類訂的,所以收穫型態若分不清,MRL 對照必然錯。PR #25 的型態消歧是本功能的必要前置,已完成。

## 二、標示設計

### 措辭原則(重要)
不使用「高風險」「危險」「安全」等**判斷性**字眼。工具沒有能力評估「被抽驗到的機率」,那是農友自己的決策。工具只陳述**可查證的法規狀態與後果**。

| 不採用 | 採用 | 理由 |
|---|---|---|
| 高風險藥劑 | **未訂殘留容許量・不得檢出** | 前者是判斷,後者是法規事實 |
| 危險 / 慎用 | **抽驗檢出即不符規定** | 陳述後果,不下指令 |
| 安全藥劑 | **已訂殘留容許量** | 「安全」是承諾,承諾不起 |

### 三級標示
| 標示 | 條件 | 說明文字 |
|---|---|---|
| 🟢 已訂殘留容許量 | 該「有效成分×農產品分類」查得到容許量 | 顯示容許量數值與標準版本 |
| 🟠 未訂容許量(不得檢出) | 對照成功但表中無此組合 | 「依現行規定屬不得檢出;市場抽驗若檢出即不符規定」 |
| ⚪ 無法對照 | 成分名或分類對不上 | 「無法自動對照,請以官方公告為準」——**不得**歸入前兩級 |

完整說明文字範例:
> **殘留檢驗提醒**:「〇〇」用於「青蒜(小葉菜類)」查無衛福部殘留容許量,依現行規定屬**不得檢出**;市場抽驗若檢出即不符規定,產品可能遭銷毀。本標示依食藥署〇年〇月版標準整理,實際請以最新公告為準。

(全段不出現「風險」二字。)

## 三、資料來源與待確認

**已確認**
- 法源:《農藥殘留容許量標準》(全國法規資料庫),「表中未列者均不得檢出」。
- 食藥署設有開放資料平台 `data.fda.gov.tw`,提供符合 OAS 標準之 API。

**待確認(實作前必須完成)**
1. MRL 標準表是否有**結構化**開放資料集(或僅法規附表 PDF/HTML,需另行解析)。
2. 農產品**分類對照表**(作物 → 小葉菜類/根莖菜類/乾豆類…)的權威來源。
3. **有效成分名稱**在兩邊的一致性:登記資料用中文普通名(如「亞滅培」),MRL 表用法確認是否相同,對不上者一律歸「無法對照」。
4. 更新頻率與版本標示方式(MRL 會修正,需可追溯版本)。

> 現行 App 的藥劑資料**完全沒有** MRL 欄位(2026-07 版全庫掃描:0 筆含「容許量/不得檢出」字樣),因此 MRL 必須以**獨立資料管線**引入,與農藥登記資料分開維護、分開標示版本。

## 四、實作分期
1. **前置(已完成)**:收穫型態消歧(PR #25)——確保作物能對到正確的農產品分類。
2. **資料層**:建立 MRL 資料集與分類對照,獨立檔案與版本號;寫入可重跑的比對測試。
3. **比對層**:有效成分 × 農產品分類 → 三級判定;對不上一律「無法對照」。
4. **顯示層**:藥劑卡片加標示與說明;先以 **beta** 標記,限已驗證的類別(蒜、豆類優先——已有真實案例)。
5. **驗證**:用虎豆案例回頭驗證判定正確;請農友種子測試者實測一輪再擴大。

## 五、建議新增的安全規則(實作時併入交接文件)
1. MRL 對照不到的組合**不得**標示為「已訂容許量」,一律顯示「無法對照」。
2. **不得**使用「高風險/危險/安全」等判斷性字眼描述殘留狀態;僅陳述法規狀態與後果。
3. MRL 標示必須附**標準版本與日期**,且與農藥登記資料版本分開顯示。
4. MRL 資料未載入或對照失敗時,**不得**顯示任何綠色(已訂容許量)標示。

## 六、已知限制
- 本功能只能反映**標準表的靜態狀態**,無法預測實際檢驗結果(受施用量、間隔、天候、代謝影響)。
- 「已訂容許量」不等於「一定驗得過」——仍須依標示劑量與安全採收期使用。
- 加工分裝流通階段的判定不在範圍內。
