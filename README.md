# 噴前查

台灣農友的農藥登記查詢、配藥計算與田間作業紀錄工具。

正式網站：https://xiao68738-hash.github.io/searchbefore/

## 新手版工作方式

1. `main` 是正式網站，不直接修改。
2. 每次工作從 `main` 建立一個分支，例如 `fix/safety-update`。
3. 在分支修改並執行測試。
4. 建立 Pull Request，確認後才合併回 `main`。
5. 合併完成並確認網站正常後，刪除已完成的分支。

## 執行安全測試

不需要安裝額外套件，只要電腦有 Node.js：

```text
node tests/run-all.js
```

## 帳號與自願贊助（v1.9.0）

- Google 登入為選用功能，目前只負責帳號識別，田區與農務資料仍保存在裝置。
- 公開測試期間不顯示試用申請、授權碼、方案或購買入口，現有功能全部開放。
- 自願贊助不影響任何功能；填入 `service-config.js` 的 HTTPS 贊助網址後才顯示可點擊連結。
- 設定步驟與安全界線見 `docs/帳號與付款設定.md`。

## 首頁功能導覽（v1.6.0）

- 首頁用一句話說明產品價值，並保留「開始查作物」的直接入口。
- 以六步流程呈現「選田區 → 查合法藥劑 → 計算用量 → 留下紀錄 → 守安全採收 → 整理匯出」。
- 以六張功能卡整理合法用藥、配藥換算、田區時間軸、安全採收、農務紀錄與備份匯出。
- 明確標示資料來源、隱私方式與使用界線，避免把自主整理檔誤認為官方直接匯入格式。

## 田間作業紀錄（v1.5.0）

- 以田區／種植批次整合用藥與其他田間作業。
- 支援栽培作業、施肥、採收、採後處理、資材購入五類紀錄。
- 田區詳情依日期合併顯示用藥與五類農務，並提供各類紀錄筆數概況。
- 新增採收紀錄時，會連動同田區全部用藥紀錄；若尚未到安全日或資料不足，儲存前明確警告並保存當時的判定快照。
- 可分別匯出農務 CSV，也可匯出用藥與農務合併的完整作業 CSV；尚未宣稱為農糧署官方匯入格式。
- 個人頁可下載完整 JSON 備份並還原。
- 資料格式由 `farm-records.js` 管理，測試位於 `tests/farm-records.test.js`。

## 發布前檢查

- 粉劑只顯示公克，不得換算瓶蓋或湯匙。
- 採收備註中的設施栽培天數不得被忽略。
- 同一田區或種植批次的所有施藥紀錄都要納入，取最晚可採日。
- 任一紀錄缺少採收期時，不得顯示「可採收」。
- 採收連動警告不得阻止已發生事件的留存，但必須要求使用者確認並保存判定狀態。
- 田間作業匯出必須標明是自主整理格式，官方規格確認前不得宣稱可直接匯入。
- 新增或調整資料欄位時，舊的本機紀錄與備份檔仍須能讀取。
- 更新 `index.html` 後同步更新 `sw.js` 的 `CACHE_VERSION`。
- Firebase 公開設定只放在 `service-config.js`；服務帳戶 JSON、管理員私鑰與金流密鑰不得上傳。
- `private_key.pem`、其他 `.pem` 私鑰檔不得上傳。

## 重要安全聲明

本工具僅供查詢與自主紀錄。實際農藥使用、稀釋倍數與安全採收期，仍以產品標示及主管機關最新公告為準。
