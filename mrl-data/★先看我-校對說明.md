# MRL 人工校對：先看這份

程式能自動判定的都做完了。這裡是**需要你判斷**的部分。

---

## 只有兩份主檔要看

| 檔案 | 內容 | 需你判斷 |
|---|---|---|
| **`殘留物對照總表.csv`** | 120 種藥劑，依證據強度排序 | **74 種** |
| **`作物分類對照.csv`** | 287 種作物 | **44 種** |

其餘 7 份是明細，**主檔看不懂時再翻**。

---

## 殘留物對照總表怎麼看

「證據層級」欄由強到弱：

| 層級 | 意思 | 要不要看 |
|---|---|---|
| 法規原文 | 附表一備註明文規定 | 掃過即可 |
| 中文通用名 | 許可證與 MRL 用同一個官方中文名 | 掃過即可 |
| 免訂容許量 | 列於免訂清單 | 掃過即可 |
| **不可視為免訂** | 該菌種逐菌株列舉，本菌株不在其中 | **要看**（3 種） |
| 疑似混合劑 | 多成分組合，部分成分對不上 | **要看**（25 種） |
| 生物製劑／鹽類 | 需確認是否等同母體 | **要看**（8 種） |
| **無線索** | 中英文名兩邊都查不到 | **要看**（41 種） |

**建議從「無線索 41 種」開始**——那是剩餘工作量的一半，而且多數可能只是名稱異寫。

## 作物分類對照怎麼看

| 分層 | 數量 | 要不要看 |
|---|---|---|
| 直接命中／名稱對應／分組對應 | 204 | 掃過即可 |
| 非農產品／觀賞花卉 | 39 | 確認沒有誤標成非食用 |
| **跨類別** | **7** | **要看**：全是豆類，乾豆類 vs 豆菜類依收穫型態而定 |
| **無法判定** | **37** | **要看**：多為中藥材與特用作物 |

跨類別那 7 種（豌豆、菜豆、花豆、豇豆、樹豆、蠶豆、鷹嘴豆）中，**有 6 種你的 App 已有「收穫型態消歧」功能可承接**，只有鷹嘴豆還沒有。

## 真的查不到的怎麼辦

`詢問函-防檢署.txt` 與 `詢問函-食藥署.txt` 已經產生，送件方式見 `詢問函-送件說明.md`。

**送出前記得刪掉你自己能確認的項目**——問題越精準，回覆越有用。

---

## 怎麼填

每份 CSV 最後都有「人工確認(是/否)」欄，部分還有「修正為」欄。

- 同意程式的判定 → 填「是」
- 不同意 → 填「否」，並在「修正為」欄寫正確答案
- 不確定 → 留空（留空的一律維持「無法確認」，App 不會顯示）

**留空是安全的。** 未確認的項目 App 一律顯示「尚未完成對照」，不會誤導農友。

## 這些檔案怎麼重新產生

```bash
cd mrl-data
node build-crop-mapping.mjs        # 作物分類
node match-exempt.mjs              # 免訂容許量
node match-residue-zh.mjs          # 中文名比對
node match-residue-en.mjs          # 英文名比對
node apply-residue-rules.mjs       # 法規原文規則
node decompose-mixtures.mjs        # 混合劑拆解
node build-residue-master.mjs      # 合併總表（最後跑）
node build-inquiry-letters.mjs     # 詢問函
```

順序有相依性，`build-residue-master.mjs` 要最後跑。
